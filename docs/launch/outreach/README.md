# Pre-launch outreach drafts

> Master prompt §6.3 deliverable. **NOT sent.** These are templates for the project maintainer to personalize, attribute, and send 7–10 days before launch.

## Purpose

To get methodological pushback from prominent skill authors *before* the launch and the workshop paper, not after. Reduces the risk of someone feeling ambushed at launch; possibly earns an endorsement or RT.

## Discipline for sending

- Personalize each one. Don't bulk-send.
- Attach a link to `docs/methodology.md` (the public URL once the repo is public).
- Acknowledge that their skill bundle will be a reference baseline.
- Invite concerns *before* publication.
- Sign as a human, not as "the agenteval project."
- Don't ask for help; ask for review.
