# Twitter — pre-launch teaser

**Send:** ~2 weeks before launch.
**Tone:** quiet, methodology-first; no marketing language. The aim is to plant the question, not to pitch the answer.

---

## Option A (recommended) — the question framing

> I've been building a reproducible benchmark for Claude Code skills.
>
> Anecdotal claims like "95% reliability vs. 60-70%" are everywhere — with zero defined task population, no control, no CIs.
>
> Early numbers are interesting. Posting the full methodology + leaderboard in ~2 weeks.

Reasoning: leads with the gap, names the claim style precisely, sets expectation for what's coming. Zero hype words.

## Option B — the receipt framing

> Spent the last 8 weeks on something concrete: an open-source eval harness for `.claude/skills/` directories.
>
> Content-addressed leaderboard, canonical 5-seed runs, two-VM re-verification, no scalar rank.
>
> Methodology doc is denser than the code. Launching in ~2 weeks.

Reasoning: more "show, not tell" — describes what was built rather than what's wrong with the status quo. Less confrontational; less viral.

## Notes

- **Don't quote a specific number yet.** A teaser with "skill X improves by 6 points" forces a defense of that number before the methodology is public.
- **Don't name a specific skill author.** The outreach emails (`docs/launch/outreach/`) go out first; the public teaser comes after they've had a chance to push back.
- **Don't tag Anthropic.** Wait for launch day for that.
- A quote-tweet of an existing "skills are amazing" thread is acceptable — frame as "what would it mean to actually measure this?". Don't be snarky.
- Set a calendar reminder for the 1-week-out thread.
